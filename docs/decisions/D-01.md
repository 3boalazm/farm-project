# D-01: Weight Source of Truth

**Status:** APPROVED, IMPLEMENTED (Wave A, 6/6 commits)

**Decision:** `animals/{animalId}/weights` is the sole canonical weight storage. All writers converge onto it.

**Context:** Weight existed in three uncoordinated locations (`animals/{id}/weights` — the real, live feature; `weight_log` — written but never read; `animals.current_weight` — written by three unrelated paths, never synchronized).

**Consequences:** Zero user-facing change for the real feature (it's already what worked); `weight_log`'s two writers required code changes; `current_weight` gained real synchronization for the first time.

**Historical Strategy:** A migration utility (`migrateWeightLogToWeights()`) was built and proven correct against simulated data, using existence-check duplicate prevention (idempotent, proven via live re-run). **Never executed against real production data** — no environment used across this project had Firebase network access.

**Implementation:** Wave A, Commits 1–6. See `docs/certification/WEIGHT.md`.
